Evaluation criteria — 5 dimensions
Specificity Depth
Looks for falsifiable details: named tools, specific numbers, team sizes, dates, and performance metrics. Vague language ("we used AI to improve things") scores low; named systems with measurable outcomes score high.
Tradeoff Authenticity
Looks for what the candidate chose not to build, with a real reason attached. Generic "we prioritised speed" scores low; "we dropped the re-ranking layer because latency SLA was 200ms and it added 80ms" scores high.
Failure Texture
Looks for a real failure arc: wrong first hypothesis, specific friction, how they diagnosed it, and what actually fixed it. Smooth success narratives score low regardless of how impressive they sound.
Constraint Proactivity
Looks for the candidate naming their own limitations — knowledge gaps, resource constraints, model behaviour they couldn't control — without being asked. Self-awareness of limits is a strong signal of genuine experience.
Problem Interrogation
Looks for the candidate questioning the problem before answering: reframing scope, surfacing unstated assumptions, or clarifying what success means. Jumping straight to a solution without interrogating it scores low.
How evaluation works — two-stage process
Stage 1
Evidence extraction. The model reads the candidate response and maps only what is explicitly stated into six categories: specific claims, named tools, failure mentions, tradeoff language, limitation acknowledgments, and problem questioning. No inference — only what is written.
Stage 2
Dimension scoring. A second model call scores all five dimensions using only the evidence map from Stage 1. It cannot score from scratch — it must work from extracted evidence and cite a verbatim quote for each score.
Why two stages? A single-pass model can fabricate plausible-sounding evidence when scoring, especially for articulate responses. Separating extraction from scoring forces the model to work from an evidence anchor, not from impressions. Low confidence is assigned when evidence is absent — not when the model is uncertain.
